Multi Functional Robot

Autonomous Line Following Mobile Robot with Obstacle Detection and Pick-and-Place Robotic Arm

Integrated Mobile Navigation and Robotic Manipulation System

Introduction Novelty Hardware Kinematics Simulation GitHub Contributors

1. Introduction

This project presents an autonomous robotic system that integrates mobile navigation and robotic manipulation. The robot follows a predefined path, detects obstacles in real time, removes them using a 4-DOF robotic arm, and resumes navigation without human intervention.

2. Novelty of the Project

  • Integration of autonomous navigation and robotic manipulation.
  • Real-time obstacle removal instead of simple avoidance.
  • Automatic switching between navigation and manipulation modes.
  • Application of kinematics concepts in practical hardware implementation.
  • Low-cost embedded robotic system design.

The physical system consists of a differential drive mobile platform integrated with a 4-DOF robotic arm. The robot is powered using a Li-ion battery and controlled by an Arduino microcontroller.

Line Following Implementation

In hardware, line following is implemented using infrared (IR) line sensors mounted at the front bottom of the robot.

These sensors detect the difference between the black line and the white surface. When the sensor detects black, it produces a different signal compared to white.

  • If the line shifts to the left, motor speeds are adjusted to turn left,where here right wheel rotates fast and left wheel rotate slow.
  • If the line shifts to the right, motor speeds are adjusted to turn right,where here left wheel rotates fast and right wheel rotate slow.
  • If both sensors detect the line, the robot moves forward.

The Arduino processes sensor readings continuously and adjusts the DC motor speeds through the L293D motor driver, allowing real-time path correction.

Obstacle Detection

Obstacle detection in hardware is implemented using an ultrasonic sensor mounted at the front of the robot.

The sensor sends ultrasonic pulses and measures the reflected echo time to calculate the distance of objects in front of the robot.

  • If the detected distance is greater than the threshold, the robot continues.
  • If the detected distance is less than the threshold, the robot stops.

When an obstacle is detected within the defined range, the control logic switches from navigation mode to manipulation mode.

Pick-and-Place Operation

The robotic arm is mounted on the mobile platform and controlled using servo motors driven by the PCA9685 servo driver.

Once the obstacle is detected and the robot stops:

  • The base servo rotates toward the object.
  • The shoulder and elbow servos extend the arm.
  • The wrist adjusts the gripping angle.
  • The gripper closes to hold the object.
  • The arm lifts the object.
  • The arm rotates and places the object outside the line path.

After successful placement, the arm returns to its initial position, and the robot resumes line-following operation.

This integration of sensing, motor control, and coordinated servo motion enables complete autonomous operation in the physical system.

4. Forward Kinematics and Inverse Kinematics

Forward Kinematics (FK)

Forward Kinematics explains how the position of the robotic arm’s gripper is determined based on the angles of its joints. In our 4-DOF robotic arm, each servo motor controls one joint: base rotation, shoulder movement, elbow extension, and wrist alignment.

When angle values are sent to these servos, the arm moves accordingly. The final position of the gripper depends on the combined motion of all four joints working together.

  • The base joint determines horizontal direction.
  • The shoulder controls vertical lifting.
  • The elbow adjusts forward reach.
  • The wrist aligns the gripper orientation.

By the practical implementation, we can observed how changing one joint affects the overall arm position. By gradually adjusting servo angle limits and testing movements, We can ensured smooth motion and safe operation of the arm.

Forward Kinematics helped us understand how multi-joint robotic arms move in three-dimensional space and how coordinated control is required for accurate positioning.

Inverse Kinematics (IK)

Inverse Kinematics is applied when the robotic arm needs to reach a specific object location. Instead of starting with joint angles, we begin with the target position of the object.

When the ultrasonic sensor detects an obstacle in front of the robot:

  • The mobile platform stops.
  • The control logic switches to arm control mode.
  • The arm begins a coordinated movement sequence toward the object.

To reach the object, the joints are moved in a structured manner:

  • The base rotates toward the object direction.
  • The shoulder lifts to the required height.
  • The elbow extends to reach the object distance.
  • The wrist adjusts for proper gripping alignment.
  • The gripper closes to securely hold the object.

After picking the object, another coordinated motion sequence moves the arm to a placement position outside the navigation path.

By applying Inverse Kinematics concepts in this practical manner, the system achieves repeatable and controlled pick-and-place operation without random or unstable movements.

5. Simulation – Navigation, Obstacle Handling and Pick-and-Place

Line Following Technique

The robot follows the path using a vision-based line tracking method. A forward-facing camera continuously captures images of the track.

The captured image is converted into grayscale and thresholded to clearly separate the black line from the background. Using image moment calculations, the centroid of the detected line is computed.

The horizontal difference between the line centroid and the image center is treated as an error value. This error is corrected using a Proportional–Derivative (PD) controller, which adjusts the steering angle smoothly.

Based on the corrected error, wheel velocities are adjusted dynamically, allowing the robot to continuously align itself with the track.

Techniques Used:

  • Image processing (Grayscale + Thresholding)
  • Centroid detection using image moments
  • PD feedback control
  • Velocity-based motion control

Obstacle Detection Technique

Obstacle detection is implemented using a ray-based distance sensing method, similar to an ultrasonic sensor.

A virtual ray is projected forward from the robot. If the ray detects an object within a predefined distance threshold, the robot immediately stops and switches to manipulation mode.

The overall system behavior is controlled using a Finite State Machine (FSM), which manages transitions between different operational states.

FSM States:

  • Line Following
  • Obstacle Detection
  • Arm Activation
  • Grasping
  • Object Placement
  • Return to Navigation

Techniques Used:

  • Ray casting for distance measurement
  • Threshold-based obstacle detection
  • Reactive control logic
  • Finite State Machine (FSM)

Pick-and-Place Operation

Once an obstacle is detected and confirmed, the robot transitions from navigation mode to manipulation mode.

The robotic arm performs a structured pick-and-place sequence:

  • The base rotates to align with the object.
  • The shoulder and elbow extend toward the target position.
  • The wrist adjusts orientation for proper gripping.
  • The gripper closes to grasp the object securely.
  • The arm lifts the object to a safe height.
  • The arm moves to a predefined placement location.
  • The gripper opens to release the object.

After the object is successfully placed outside the navigation path, the system transitions back to line-following mode and resumes autonomous movement.

This coordinated manipulation sequence demonstrates integration between perception, decision-making, and robotic arm control.
